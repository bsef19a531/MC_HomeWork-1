GitHub is a code hosting platform for collaboration and version control. GitHub lets you (and others) work together on projects.

GitHub essentials are:

  • Repositories
  • Branches
  • Commit
  • Git (The version Control Software, Github is built on)

In order to host your Repositories online on github you need to install git on your machine and signup on github platform.

  • How to Push a Repository on Github

  • Signin to GitHub:

Click the link above to visit Github website. You will observe the page as shown in image below.

  • Click the signin button.
  • If you are not already registered click signup button.

  • Creating a new Repository:

  • After logging into the platform click the button at top-right corner and select the option to create new Repository.
  • You will be directed to a form as shown below.
  • Enter a name for the Repository you want to create
  • Enter a description about your Repository if you want (Optional)
  • You can choose to make your Repository either to be Public or Private
  • You can choose to initialize your repository with a detailed README document
  • After choosing all required options, click on create repository button
  • Then, you will be redirected to the page shown below.
  • You can choose Either Https or SSH method to host your code.
  • You will be given three options where you can select to either create a new repository by using command line, This can be done by running the command given in the section on the gitbash commandline.
  • Second You can push an existing repository from your local machine to Github, The procedure is similar just paste the commands given in section in gitbash commandline.
  • Third you can import can import code from another hosted repository by following the same procedure.
  • You are all set now, by following any of the method your desired repository will be created.

  • Basic Important Git Commands:

Here are some important git command which are used to push files to github.

  • sit status - to check the status of the repository (Git repository or not and files which are being tracked)
  • git init - to initialize a git repository
  • git add --all - to add all files in git repository for stagging
  • git commit -a -m "Type msg here" - to commit all the stagged files with a short message
  • git log - to check the details of commits done on a git repository
  • git remote - to check the linked remote repository with the local repository
  • git push -u origin master - to push git repository code to the master branch of the remote repository
